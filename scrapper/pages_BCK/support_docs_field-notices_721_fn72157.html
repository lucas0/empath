Problem Description A limited number of NCS1K-2KW-DC, Power Supply Units (PSUs), with HW Version VID 02, shipped with an incorrect Firmware might report the FW upgrade flag continuously. Background During manufacturing, a limited number of the PSUs with Hardware (HW) Version ID (VID) 02 were incorrectly loaded with the FW Version 1.14, instead of FW Version 2.01. This will cause FW mismatch alarm. Cisco IOS® XR releases 7.0.1 and later verifies the PSU VID 01 with FW 1.14 and VID 02 with FW 2.01. The FW mismatch alarm will be raised if the PSU VID and FW version are not same as mentioned above. Software Releases earlier than 7.0.1 do not recognize FW Version 2.01, so they do not raise any alarms. Problem Symptom Cisco IOS XR software detects FW compatibility via the TAN HW version. After a reload with the admin hw-module location all reload command and a reboot, the status changes back to "NEED UPGD". In Summary: Cisco IOS XR Software Releases 7.0.1 and later do not allow you to upgrade the PSU. It goes to "UPGD SKIP" or "RLOAD REQ" and if reloaded, goes back to the "NEED UPGD" state. Cisco IOS XR releases earlier than Software Release 7.0.1 do not detect any inconsistency. Workaround/Solution For the limited number of PSUs which show the problem symptoms, and if the serial number shows Affected in the Serial Number Validation Tool, Cisco recommends to replace the affected PSU.
