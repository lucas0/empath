Prerequisites Before you start, ensure that you have the: Active Cisco.com account Smart account administrator user role Add New Users to a Smart Account Manually Step 1. Navigate to Cisco Software Central and log in with your Cisco.com account as shown in the image. Step 2. In the Manage Smart Account section, click Manage account as shown in the image. Step 3. On the Users tab, click Users, and then Add Users.... Step result: The system displays the Add Users page as shown in the image. Step 4. On the Add Users page, on the Select Users tab, from the Method drop-down list, select Add Users Method, and then in the Users to Add field, enter Cisco user IDs or email addresses of the users that you want to add. Step result: The system displays the list of users that you added. Note: In the Method drop-down list, you also have the option to import new users from a CSV file. Step 5. When you have added new users, click Next. Step result: The system displays the Assign Accountstab. Note: If you add an incorrect user by mistake, click Remove in the Actions column. Step 6. On the Assign Accounts tab, in the Assign Accounts to the Users section: From Scope of Access, select one of these options: From the User Role drop-down list, select: a. Entire Smart Account The user has access to the entire Smart account, which includes all the Virtual accounts. Option 1: Smart Account Approver Smart account approvers can edit Smart account properties, view all users, accept agreements, and view event logs. Approvers cannot manage licenses and they do not have the privilege to approve or decline an access request. Option 2: Smart Account Administrator Smart account administrators can edit Smart account properties, add and edit users and Virtual accounts, accept agreements, view event logs, and manage licenses for the entire Smart account. Only a Smart account administrator can approve or decline an access request. Option 3: Smart Account User Smart account users can access all Virtual accounts and perform licensing activities but cannot create new Virtual accounts or manage users. Click Next. b. Selected Virtual Accounts The user has access to the selected Virtual accounts only. Caution: The system displays the available Virtual accounts that you can assign to the user only after you select the user role first. Option 1: Virtual Account Administrator Virtual account administrators can manually add and edit users to the assigned Virtual accounts, view event logs for the assigned Virtual accounts, view account agreements, and manage licenses for the assigned Virtual accounts. Option 2: Virtual Account User Virtual account users can manage licensing for Virtual accounts that they are assigned to, but they cannot add new users. In the Available Virtual Accounts section, select the check box next to the Virtual account that you want to assign to new users, and then click Assign Account and Children to assign a specific Virtual account and its nested Virtual accounts or Assign Account Onlyto assign a specific Virtual account. The system displays the assigned Virtual account in the Assigned Virtual Accounts section. Click Next. Note: To unassign Virtual accounts or Virtual accounts and their nested Virtual accounts from the Assigned Virtual Accounts section, select the check box next to the Virtual account, and then click the Unassign Selected or Unassign All button. Step result: The system displays the Review tab as shown in the image. Step 7. On the Reviewtab, ensure that the information in the Users and Account Access sections are correct and then click Add Users. Step result: The system displays the Users page with the new users added to the list as shown in the image. Troubleshoot If you experience an issue when you manually add new users to a Smart account, open a case in Support Case Manager (SCM).
