Background When Perl version 5.30.1a is already installed on the server (by SWManager versions 914409 to 945177), patch ap383514 does not install the new Perl upgrade which has the same version 5.30.1a. Consequently, BroadWorks fails to start. Problem Symptom After the patch is applied, the command startbw (used to start BroadWorks) fails with this error: $ startbw BroadWorks control script version 23.0 Can't locate object method "start_SSL" via package "IO::Socket::SSL" at /usr/local/broadworks/AS_Rel_23.0_1.1075/bin/bcctConnection.pm line 214. Workaround/Solution Workaround Remove patch ap383514. Solution In order to resolve this issue, customers should download and run corrective patch ap384639 which is available from Cisco Software Central: BroadWorks Application Server - AP.platform.23.0.1075.ap384639 BroadWorks Profile Server - AP.platform.23.0.1075.ap384639 BroadWorks Network Server - AP.platform.23.0.1075.ap384639 BroadWorks Xtended Services Platform - AP.platform.23.0.1075.ap384639 For More Information If you require further assistance, or if you have any further questions regarding this field notice, please contact the Cisco Systems Technical Assistance Center (TAC) by one of the following methods: Open a service request on Cisco.com By email or telephone Receive Email Notification For New Field Notices My Notificationsâ€”Set up a profile to receive email updates about reliability, safety, network security, and end-of-sale issues for the Cisco products you specify.
