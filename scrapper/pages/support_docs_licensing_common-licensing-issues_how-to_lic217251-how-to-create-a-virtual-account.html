Virtual account is a subfolder in a Smart account that helps you manage your licenses, devices, services, and subscriptions. There is no limit to the number of Virtual accounts that you can have in your Smart account. However, you can have a maximum of ten subfolder layers in each Virtual account. When a Smart account is created, a default Virtual account is automatically created. Virtual accounts enable you to organize and manage your devices and licenses; grant management rights to the users within a Virtual account; control access and assign users to specific accounts; and categorize licenses and devices by your organizational needs. Prerequisites Before you start, make sure you have the following: Active Cisco.com account Smart account administrator role (To request access to a Smart account or Virtual account, refer to How to Request Access to an Existing Smart Account.) Creating a Virtual Account Step 1: Go to Cisco Software Central and log in with your Cisco.com account. Step 2: In the Manage Smart Account section, click Manage account. Step 3: On the Virtual Accounts tab, click Create Virtual Account. Step result: The system displays the Create New Virtual Account page. Step 4: On the Create New Virtual Account page, on the Account Settings tab, fill in the Name field. From the Parent Account drop-down list, select whether you want to place the new Virtual account beneath an existing Virtual account that you have an access to, or place it at the top level - directly beneath the Smart account. From the Access Level drop-down list, select whether you want to keep the new Virtual account public or private. Note: By default, the system places the new Virtual account at the top level. Caution: If the new Virtual account is visible in commerce applications, it simplifies depositing of licenses during purchase. By setting the Access Level to Private, users cannot find this Virtual account in the search results of the Cisco Commerce Workspace. By setting the Access Level to Public, the users can find this Virtual account in the search results of Cisco Commerce Workspace. Step 5: On the Assign Users tab: Click: From the following drop-down lists: Select one of the following options: The Assign Users button if you need to assign more users during the Virtual account creation process. Access Scope Option 1: This Account only The user has access only to a selected Virtual account. Option 2: This Account and its Children The user has access to both the selected Virtual account and subfolder Virtual account within the main one. User Role Option 1: Virtual Account Administrator Virtual account administrators can add, edit, or delete subfolders to their respective Virtual accounts. Option 2: Virtual Account User Virtual account users cannot add new users or create subfolders. They can only access licenses within their assigned Virtual account. Method Option 1: Add Users Manually The system displays the Users to Add field, in which you can enter the individual users' IDs or email addresses, and these users will then be able to access the Virtual account. Option 2: Import from CSV File The system displays the Upload Users button that enables you to upload the CSV file. After selecting the options that accommodate your business needs, click OK. Step result: The system returns to the Assign Users tab. Click Next. b. The Next button if you want to skip adding new users at this point in time. You can always add users after the Virtual account creation process by clicking the Users tab on the menu bar. Note: Some users are going to have access to this Virtual account because they inherit access from a parent account. Step 6: On the Review tab, review the Virtual account settings, and then click Create Virtual Account. Step result: The system displays a notification message that confirms the successful creation of the new Virtual account. Troubleshooting If you experience an issue during the Virtual account creation, open a case inSupport Case Manager (SCM).
