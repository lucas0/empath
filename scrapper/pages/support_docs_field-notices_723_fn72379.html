Background Cisco BroadWorks NDS, NFM, and UMS use MariaDB as their database management system. MariaDB 10.2 will be end-of-life (EOL) on May 23, 2022. It is important to upgrade to MariaDB 10.3 in order to continue to receive support and fixes. NDS and NFM Release 2020.07 and earlier use MariaDB 10.2. NDS and NFM Release 2020.08 and later use MariaDB 10.3. Problem Symptom You will not be able to receive support and fixes without an upgrade to MariaDB 10.3. Workaround/Solution Workaround In BroadWorks Release 22.0, these patches provide MariaDB 10.3.34a: Network Database Server AP.nds.22.0.1123.ap374251 Follow the patch instructions in order to upgrade MariaDB. Unified Messaging Server AP.ums.22.0.1123.ap374251 Follow the patch instructions in order to upgrade MariaDB. Network Function Manager No patch available, upgrade to the latest version shown in the Solution section. Solution Customers should plan to upgrade to the latest release of the NDS and NFM: Network Database Server NDS_Rel_2022.02_1.193.Linux-x86_64.bin Network Function Manager NFM_Rel_2022.02_1.193.Linux-x86_64.bin Unified Messaging Server Upgrade not available, install the patch in the Workaround section.
