Background Some versions of FMC software might cause the security management platform to halt unexpectedly. The Sybase database becomes unresponsive, which causes the issue. Problem Symptom The FMC appliance halts and indicates "System processes are starting" over the console port. When you switch to expert mode in the CLI, this message is displayed: You have logged in while system startup is in progress. Please wait, some feature may be unavailable until startup is complete. The web-based GUI shows this message: The server response was not understood. Please contact support. The Sybase database located in /var/opt/CSCOpx/MDC/log/operation/sydb.out will contain logs similar to these with timestamps that match the onset of the issue. >W. 11/27 23:40:41. Task 0x19b66b0(Request task 76) is trying get forbid mutex held by task 0x19b5d10(Request task 69) for more than 60000 ms >W. 11/27 23:40:41. Task 0x19b5d10(Request task 69) state: worker=0x1549e0003db0 wakend=0 waiting=40 >W. 11/27 23:40:41. Worker 0x1549e0003db0 state: wakend=0 waiting=0 in_atomic=0 is_dedicated=0 in_critical_section=0 allow_forbid=0 Workaround/Solution Workaround In order to restart the FMC database with the CLI (expert mode), enter these commands: fmc> expert fmc$ sudo su fmc# pmtool restartbyid vmsDbEngine Solution Cisco recommends that you upgrade the FMC software to one of these versions in order to resolve the issue.
