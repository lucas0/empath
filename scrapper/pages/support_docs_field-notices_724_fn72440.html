Background After patch AP382053 is applied and the Enhanced XSI Mechanism is activated, the Open Client Interface-Call Control (OCI-C) subscription could be rejected with this response: Unauthorized request oci:errorCode 100004 Problem Symptom When patch AP382053 is applied and the new feature, Enhanced XSI Mechanism is activated (Feature ID 101642), these two problems can occur: A ClassCastException can occur and this signature can be seen in the logs: java.lang.ClassCastException: com.broadsoft.protocols.ocic.OCICEventsUtilityDb$GetAdminUserIdByUIDQuery cannot be cast to com.broadsoft.protocols.ocic.OCICEventsUtilityDb$GetUserFromBlfUriQuery If the Busy Lamp Field (BLF) Uniform Resource Identifier (URI) uses the same URI as an existing user, the OCI-C subscription could be rejected with oci:ErrorResponse (Unauthorized request oci:errorCode 100004). Workaround/Solution For the first problem described in the Problem Symptom section, install the corrective application server (AS) patch AP383459, which is available on the Cisco Software Download page: Release 22.0: AP.as.22.0.1123.ap383459 Release 23.0: AP.as.23.0.1075.ap383459 Release 24.0: AP.as.24.0.944.ap383459 For the second problem described in the Problem Symptom section, install the corrective AS patch AP383520, which is available on the Cisco Software Download page: Release 22.0: AP.as.22.0.1123.ap383520 Release 23.0: AP.as.23.0.1075.ap383520 Release 24.0: AP.as.24.0.944.ap383520 For More Information If you require further assistance, or if you have any further questions regarding this field notice, please contact the Cisco Systems Technical Assistance Center (TAC) by one of the following methods: Open a service request on Cisco.com By email or telephone Receive Email Notification For New Field Notices My Notificationsâ€”Set up a profile to receive email updates about reliability, safety, network security, and end-of-sale issues for the Cisco products you specify.
