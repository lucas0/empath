Background A traceback and reload condition for the Cisco Secure Firewall 2100 Series security appliances might occur for applications that use the App-Cache process. The traceback and reload condition is due to a memory management issue that might occur for these applications. In order to display a summary of applications that use the App-Cache process, enter the CLI command show memory app-cache. See the Cisco Secure Firewall ASA Series Command Reference or the Cisco Secure Firewall Threat Defense Command Reference for additional information. Problem Symptom The Cisco Secure Firewall 2100 Series security appliance reloads and indicates thread name 'lina' as the faulting thread. Workaround/Solution Cisco recommends to upgrade the ASA or Firepower software in order to fix the issue. For ASA-based devices, upgrade to one of the Cisco ASA software versions shown in this table in order to resolve the traceback and reload issue.
