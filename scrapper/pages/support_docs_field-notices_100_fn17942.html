Products Affected Product Top Assembly Part Number NCB/15252= 800-09474-01 Serial Numbers Random # CEM05085392, CEM05085394, CEM05090008, CEM05090014, CEM05090019, CEM05090003, CEM05090011, CEM05090012, CEM05090020, CEM05100001, CEM05160050, CEM05110011, CEM05110004, CEM05110009, CEM05110003, CEM05110005, CEM05090015, CEM05110007, CEM05160058, CEM05160048, CEM05160052, SLM05190085, CEM05110008, CEM05170144, CEM05110002, CEM05090013, CEM05090009, CEM05160055, CEM05110006, CEM05110012, CEM05090018, CEM05110012 Problem Description Sbunetwork Manager (SNM) Software Release 1.1 requires a minimum network control board (NCB) level of 800-09474-02. The SNM Release 1.1 software does not support any previous NCB version. Before customers decide to upgrade to SNM Release 1.1 software, ensure that the NCB board is at the minimum level of 800-09474-02. Background A limited number of customers received NCBs with a level of 800-09474-01. This level is not compatible with SNM Release 1.1 software. To check the NCB part number and serial number log in as Administrator or Operator and use the command version. NCB:=version NCB Part No:800-09474-01 NCB Revision No: 01 NCB Serial No: CEM05090011 SNM Part No: 36A0012 SNM Version No: 1.0(3.19) NCB:= See chapter 4.12.14 of the Cisco ONS 15200 Command Line Interface Administrator Manual for more details on the version command. Problem Symptoms If your NCB has a serial number found in the above list do not upgrade the SNM software to Release 1.1. Replace the NCB with the correct version and then upgrade to the new software. The upgrade procedures can be found in the document titled How-To Replace the NCB Module on the ONS 15252. Workaround/Solution All affected customers received the upgraded NCB in October 2001. Before upgrading the software to SNM Release 1.1, make sure to use the latest NCB version.
