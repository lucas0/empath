Background When upgrading to a target release of R23.0 or R24.0, if an automatic auto-revert process occurs, the server may be left in an unusable state. Patch AP384127 fixes the auto-revert process. Problem Symptom The server, daemons, CLI and other processes will not start when sudoers is affected or when wrong third party versions are being used. Workaround/Solution Customers who upgrade to R23 or R24 should download and apply these corrective patches before attempting an upgrade: R23 AP.platform.23.0.1075.ap384127 AP.as.23.0.1075.ap384127 AP.ps.23.0.1075.ap384127 AP.ns.23.0.1075.ap384127 AP.xsp.23.0.1075.ap384127 R24 AP.as.24.0.944.ap384127 For More Information If you require further assistance, or if you have any further questions regarding this field notice, please contact the Cisco Systems Technical Assistance Center (TAC) by one of the following methods: Open a service request on Cisco.com By email or telephone Receive Email Notification For New Field Notices My Notificationsâ€”Set up a profile to receive email updates about reliability, safety, network security, and end-of-sale issues for the Cisco products you specify.
