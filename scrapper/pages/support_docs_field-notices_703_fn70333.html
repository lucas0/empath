Revision History Revision Publish Date Comments 1.0 10-Dec-18 Initial Release Products Affected Affected Product ID Comments NCS1K-2KW-DC NCS1K-2KW-DC= Part Alternate Defect Information Defect ID Headline CSCvm62177 NCS1002:DC PSU- XR 6.3.2 doesn't recognise DC PSU module PID as valid PID - Failed provisionioning Problem Description The DC power supply, NCS1K-2KW-DC in the Network Convergence System 1002 (NCS 1002) chassis is not recognized and a critical alarm is observed when the software is upgraded from Cisco IOS® XR Software Release 6.3.1 to Cisco IOS XR Software Release 6.3.2 or 6.5.1. In such SW configuration, when DC power supply is used, XR is limiting the provisioning and power to only two slices of the system. Background The issue is due to a power budget enabled with a wrong limitation. The operating power range for DC power supplies is miscategorized in the operating system and causes the unrecognized power supply error and alarm. This issue is addressed by extending the supported power range for DC power supply units. See the Workaround/Solution section in order to resolve this issue. Problem Symptom A NCS 1002 chassis equipped with DC power supply modules (NCS1K-2KW-DC) that runs Cisco IOS XR Software Release 6.3.2 or 6.5.1 shows a critical alarm (error message) as below: "Unsupported Power module unit - NCS1K-2KW-AC2". The show inventory and show diags commands on PM modules display the correct information as shown here: RP/0/RP0/CPU0:ios#sh inventory Sat Sep 22 01:53:42.469 UTC . . . NAME: "0/PM0", DESCR: "Network Convergence System 1000 2KW DC PSU" PID: NCS1K-2KW-DC , VID: V01, SN: POG2122Z02K NAME: "0/PM1", DESCR: "Network Convergence System 1000 2KW DC PSU" PID: NCS1K-2KW-DC , VID: V01, SN: POG2122Z020 RP/0/RP0/CPU0:ios# However, the show alarm brief command displays: ‘show alarms brief’ lists “Unsupported Power module unit - NCS1K-2KW-AC2” sysadmin-vm:0_RP0#sh alarms brief Sat Sep 22 01:54:23.838 UTC ------------------------------------------------------------------------------- Active Alarms ------------------------------------------------------------------------------- Location Severity Group Set time Description ------------------------------------------------------------------------------- 0/RP0 critical power 09/22/18 01:45:34 Unsupported Power module unit - NCS1K-2KW-AC2 0/RP0 critical power 09/22/18 01:45:34 Unsupported Power module unit - NCS1K-2KW-AC2 0 minor environ 09/22/18 01:45:44 Power reservation exceeds available power. Workaround/Solution Install and activate the appropriate Software Maintance Upgrade (SMU) in order to recover from the issue as mentioned in the Problem Symptom section. Based on the relevant Cisco IOS XR Software Release (6.3.2 or 6.5.1), locate the SMU with the software version link and select the necsessary SMUs based on this table. Note: A Cisco user ID and password is required.
